APRENDIZAJES

1. Para la implementación de la automatización API se utilizó Karate Framework 2.0.0, versión con la que no se tenía experiencia previa, representando una oportunidad de aprendizaje durante el desarrollo del ejercicio.
2. Durante la configuración inicial se identificaron diferencias importantes respecto a Karate 1.x, principalmente en la gestión de dependencias, paquetes y compatibilidad con versiones recientes de Java.
3. Uno de los principales retos encontrados fue la compatibilidad entre Karate 1.4.1 y Java 24. Debido a limitaciones relacionadas con dependencias internas de GraalVM, fue necesario migrar a Karate 2.0.0 para garantizar la correcta ejecución sobre Java 24.
4. Durante la migración se evidenciaron cambios en la forma de interpretar expresiones dinámicas dentro de archivos JSON y cuerpos de petición, especialmente en el uso de variables dentro de payloads, lo que requirió adaptar la construcción de algunos request para asegurar su correcta evaluación.
5. Se observó que la API pública PetStore genera automáticamente el identificador de las mascotas, por lo que la estrategia de automatización fue ajustada para reutilizar dinámicamente el ID retornado por la respuesta en las operaciones posteriores del flujo.
6. La experiencia permitió comprender las diferencias entre las versiones de Karate y la importancia de validar la compatibilidad entre framework, librerías asociadas y versión de Java utilizada antes de iniciar un proyecto de automatización.
7. Karate 2.0.0 demostró ser una alternativa moderna y compatible con versiones recientes de Java, manteniendo la simplicidad característica del framework para la automatización de servicios REST.

La solución implementada cumple satisfactoriamente los requerimientos planteados para el ejercicio técnico.
